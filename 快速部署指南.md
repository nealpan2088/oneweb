# 快速部署指南
**一句话原则：备份 → 测试 → 切换 → 验证**

## 🚀 5分钟快速部署

### 第一步：准备新版本
```bash
# 1. 创建新版本目录
cd /home/admin/yunxinwangdian/
mkdir frontend-vX  # X是版本号，如v4

# 2. 上传代码到新目录
# 通过SCP或直接复制文件到 frontend-vX/

# 3. 设置权限
chmod -R 755 frontend-vX/
```

### 第二步：配置Nginx
```bash
# 1. 备份当前配置
sudo cp /etc/nginx/conf.d/yunxinwangdian.conf /etc/nginx/conf.d/yunxinwangdian.conf.backup-$(date +%Y%m%d-%H%M%S)

# 2. 添加新版本配置
sudo sed -i '/^}$/i\    # 单店版顾客端 vX\n    location /vX/ {\n        alias /home/admin/yunxinwangdian/frontend-vX/;\n        index index.html;\n    }' /etc/nginx/conf.d/yunxinwangdian.conf

# 3. 测试配置
sudo nginx -t

# 4. 重载Nginx
sudo systemctl reload nginx
```

### 第三步：验证部署
```bash
# 测试访问
curl -I http://localhost/vX/

# 应该返回: HTTP/1.1 200 OK
```

## 🔄 切换到生产环境

### 当新版本测试通过后
```bash
# 1. 再次备份
sudo cp /etc/nginx/conf.d/yunxinwangdian.conf /etc/nginx/conf.d/yunxinwangdian.conf.backup-before-switch

# 2. 修改配置（vX成为生产，旧版本移到/v旧/）
sudo sed -i 's|location / {|location /v旧/ {\n        alias /home/admin/yunxinwangdian/frontend/;\n        index shop_index.html;\n    }\n\n    # 单店版顾客端 vX (新生产版本)\n    location / {|' /etc/nginx/conf.d/yunxinwangdian.conf
sudo sed -i 's|location /vX/ {|location / {|' /etc/nginx/conf.d/yunxinwangdian.conf
sudo sed -i 's|alias /home/admin/yunxinwangdian/frontend-vX/;|root /home/admin/yunxinwangdian/frontend-vX;|' /etc/nginx/conf.d/yunxinwangdian.conf

# 3. 应用配置
sudo nginx -t && sudo systemctl reload nginx

# 4. 验证
curl -f http://localhost/ && echo "切换成功"
```

## ⚡ 紧急回滚

### 如果新版本有问题
```bash
# 1. 恢复最近备份
sudo cp $(ls -t /etc/nginx/conf.d/yunxinwangdian.conf.backup* | head -1) /etc/nginx/conf.d/yunxinwangdian.conf

# 2. 重载Nginx
sudo nginx -t && sudo systemctl reload nginx

# 3. 验证
curl -f http://localhost/ && echo "回滚成功"
```

## 🆘 常见问题解决

### 1. 404错误
```bash
# 检查文件是否存在
ls -la /home/admin/yunxinwangdian/frontend-vX/index.html

# 检查Nginx配置
grep -n "vX" /etc/nginx/conf.d/yunxinwangdian.conf
```

### 2. 502错误
```bash
# 检查后端服务
ps aux | grep node

# 重启后端
cd /home/admin/yunxinwangdian/backend
pkill -f "node server.js"
nohup node server.js > ../logs/app.log 2>&1 &
```

### 3. 权限问题
```bash
# 修复权限
sudo chmod -R 755 /home/admin/yunxinwangdian/frontend-vX/
```

## 🎯 记住这几点

1. **永远先备份** - 操作前备份配置
2. **先测试再生产** - 用/vX/路径测试
3. **随时能回滚** - 保留备份文件
4. **监控验证** - 部署后立即检查
5. **小步快跑** - 每次只改一小部分
