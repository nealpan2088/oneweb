📶 WiFi密码功能修复测试

🔧 问题诊断

错误信息: GET http://47.110.156.11/api/shop/3/wifi 404 (Not Found)

根本原因: 前端使用GET请求，但后端只支持POST请求

📋 问题分析

后端API设计:

  • ✅ POST /api/shop/:id/wifi - 需要传递 { action: 'confirm' }
  • ❌ GET /api/shop/:id/wifi - 不存在此端点

前端代码问题:

  • 原代码: await this.apiClient.get(`/${shopId}/wifi`)
  • 修复后: await this.apiClient.post(`/${shopId}/wifi`, { action: 'confirm' })

业务逻辑:

WiFi密码是店铺的敏感信息，需要用户先确认已加微信好友，才能获取密码。

🔧 修复内容

1. 修复 shopService.js

// 修复前 - 使用GET请求（错误） async getWifiInfo(shopId = this.shopId) { const wifiInfo = await this.apiClient.get(`/${shopId}/wifi`); // ... } // 修复后 - 使用POST请求（正确） async getWifiInfo(shopId = this.shopId) { const wifiInfo = await this.apiClient.post(`/${shopId}/wifi`, { action: 'confirm' }); // ... }

2. 更新店铺页面函数

// 修复前 - 直接获取WiFi密码 async function showWifiPassword() { const wifiInfo = await shopService.getWifiInfo(); // 显示密码... } // 修复后 - 先确认用户已加微信 async function showWifiPassword() { // 先询问用户是否已加微信 const confirmed = confirm('📶 获取WiFi密码\n\n为了确保您是店铺顾客，请先确认：\n\n1. 已添加店铺微信好友\n2. 已向店员说明来意\n\n确认已完成后，点击"确定"获取WiFi密码。'); if (!confirmed) return; // 用户取消 const wifiInfo = await shopService.getWifiInfo(); // 显示密码... }

⚠️ 常见问题

Q: 为什么还需要用户确认已加微信？

A: 这是后端API的设计要求，WiFi密码是店铺的敏感信息，需要确保用户是真实顾客。

Q: 如果用户点击"取消"会怎样？

A: 函数会直接返回，不会获取WiFi密码，也不会记录访问日志。

Q: 如果API连接失败会怎样？

A: 会显示默认的WiFi信息，确保用户体验不受影响。

Q: 为什么之前能工作？

A: 可能之前有GET端点，或者使用了不同的API版本。
